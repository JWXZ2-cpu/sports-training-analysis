三、队医-教练双向协调数据流（核心机制）

背景

队医使用超声波等治疗设备时，治疗后1-2天治疗部位会非常疼痛。
如果第二天该部位有大强度训练，会影响运动员表现。
因此队医必须提前知道明日训练安排，教练也必须知道今晚治疗计划。

│
└─ 教练确认：赛前激活训练计划无需调整

比赛前1天（D-1）：
├─ 今日训练：赛前激活（轻量，各部位激活）
├─ 队医：不做治疗，只做赛前准备（贴扎、放松等）
├─ 教练：确认比赛阵容和战术

├─ 运动员上场比赛
├─ 队医：场边待命

比赛后（D+1）：
├─ 恢复训练
├─ 队医：根据比赛表现和赛后体检，制定赛后恢复治疗计划
└─ 教练：根据比赛结果和运动员状态，调整恢复期训练计划

│  （如：近两周跑量增加过快、某部位训练频率过高等）

    ↓

├─ 科研更新运动员生理指标

├─ 教练根据恢复进度逐步调整训练计划

│                  分发逻辑                             │
│                                                     │
│  1. 所有角色                                        │
│     └─ 数据入库，统计数字更新                         │
│                                                     │
│  2. 运动员本人                                       │
│     └─ 收到通知："您的训练分析报告已生成"              │
│     └─ 可查看：diary_text, recommendations,          │
│        training_zone, training_quality               │
│     └─ 不可查看：coach_summary, risk相关              │
│                                                     │
│  3. 主教练                                           │
│     └─ 全队概览中该运动员状态更新                      │
│     └─ 可查看：报告全部内容                           │
│     └─ 特别关注：risk_flag=true的报告自动置顶         │
│                                                     │
│  4. 助教                                             │
│     └─ 训练日志列表中出现新记录                        │
│     └─ 可查看：报告全部内容                           │
│     └─ 可添加：训练评价和反馈备注                      │
│                                                     │
│  5. 队医                                             │
│     └─ 如 risk_flag=true 且涉及伤病：                │
│        加入伤病预警列表，标记为高优先级                │
│     └─ 可查看：fatigue, body_parts,                  │
│        treatment相关字段                              │
│     └─ 重点关注：身体部位疼痛记录                     │
│                                                     │
│  6. 体能教练                                         │
│     └─ 如涉及力量/体能问题：收到提醒                  │
│     └─ 可查看：与体能相关的分析字段                   │
│                                                     │
│  7. 科研                                             │
│     └─ 原始数据进入统计分析池                         │
│     └─ 更新运动员长期数据趋势                         │
│                                                     │
│  8. 管理人员                                         │
│     └─ 仪表盘统计数字更新                             │
│     └─ 训练完成率、出勤率等更新                       │
└─────────────────────────────────────────────────────┘

通知存入数据库 notifications 表：

{
  "id": number,
  "recipient_id": number,           // 接收者用户ID
  "sender_id": number,              // 触发者用户ID（可为系统）
  "type": string,                   // 通知类型
  "title": string,                  // 通知标题
  "content": string,                // 通知内容
  "related_entity_type": string,    // 关联实体类型（report/injury/treatment等）
  "related_entity_id": number,      // 关联实体ID
  "priority": "normal|high|urgent", // 优先级
  "is_read": boolean,               // 是否已读
  "created_at": datetime            // 创建时间
}

├─ 主教练/助教界面右上角：通知铃铛图标 + 未读数量角标

├─ 点击通知：跳转到对应详情页面
├─ 高优先级通知：弹窗提醒
└─ 紧急通知（如严重伤病）：弹窗 + 声音提醒
